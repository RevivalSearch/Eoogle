Why We Cache Users

At Eoogle, we cache ECSR usernames and user data for several important reasons:

  • Performance: Caching reduces the number of API calls to ECSR's servers, resulting in faster response times for our users.
  • Reliability: By storing user data locally, we can provide username lookups even when ECSR's API is experiencing issues.
  • Historical Data: Caching allows us to maintain a history of username changes that would otherwise be lost.
  • Rate Limiting: It helps us respect ECSR's API rate limits by reducing the number of requests we need to make.

We take user privacy seriously. All cached data is used solely for the purpose of providing the Eoogle service. Users can request the removal of their data by contacting us.
